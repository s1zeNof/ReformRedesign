# Редизайн сторінки «Електролабораторія Nissan»

## 1. Основні проблеми поточної сторінки

**Контент і цінність**
- Hero не продає: лише назва на фото. Немає УТП (мобільна лабораторія на електротязі, без вихлопів, виїзд на місце), ключових цифр, CTA.
- Опис продукту — один абзац у блідому alert-блоці, без структури і виділених переваг.
- «Технічні характеристики» сховані в згорнутому акордеоні — головна технічна інформація невидима.
- Галерея — 14 однакових мініатюр без підписів і групування (екстер'єр / салон / обладнання / двері-відсіки). Користувач не розуміє що бачить.
- Немає пояснення сценаріїв застосування (хто замовник: екологічні служби, лабораторії, держустанови).
- Немає блоку «Комплектація / обладнання» окремо від ТХ.
- Відсутні: процес замовлення, FAQ, схожі моделі, форма заявки на сторінці (тільки footer-кнопка).

**UX / структура**
- Хлібні крихти на фото читаються погано.
- Один блок тексту = погана скануваність.
- Немає sticky CTA / форми лідогенерації біля контенту.
- Mobile: 4 колонки галереї → дрібні фото.

**Візуал**
- Жовто-синя шапка + alert із жовтою рамкою = візуальний шум, конкурує з контентом.
- Типографіка пласка, єдиний H1, далі дрібний текст.
- Немає іконок, акцентів, ритму між блоками.
- Фото в hero розмите і обрізане — найкраща частина авто за заголовком.

**SEO / технічне**
- Title = назва моделі без ключів («електролабораторія на базі електромобіля Nissan e-NV200»).
- Немає alt-описів, structured data (Product/Vehicle), OG-зображення.

## 2. Запропонована структура сторінки

```
1. Hero
   - Велике чисте фото авто (3/4 ракурс), без overlay поверх машини
   - H1 + підзаголовок-УТП + 3 ключові пункти (zero-emission, виїзд на місце, готова під ключ)
   - 2 CTA: «Отримати пропозицію» + «Завантажити PDF-специфікацію»
   - Бейдж: «Eco Mobile Lab / IFC партнер»

2. Швидкі факти (стрічка з 4 цифр/іконок)
   Запас ходу • Корисне навантаження • Час підготовки до виїзду • Кількість робочих місць

3. Призначення (для кого і навіщо)
   3 картки сценаріїв: екологічний контроль, відбір проб (повітря/вода/ґрунт), виїзні випробування

4. Опис рішення (поточний текст, переписаний у 2–3 короткі параграфи + ілюстрація)

5. Обладнання та комплектація
   Згруповані пункти з іконками (вимірювальне, відбір проб, енергоживлення, меблювання, безпека)

6. Технічні характеристики
   Розгорнута таблиця за замовчуванням, дві колонки: «Шасі» / «Надбудова»

7. Галерея з вкладками
   Вкладки: Екстер'єр · Салон · Робоча зона · Обладнання · Двері/відсіки
   Lightbox при кліку, підписи

8. Процес замовлення (4 кроки)
   Брифінг → ТЗ і КП → Виробництво → Передача і навчання

9. FAQ (4–6 питань: терміни, гарантія, кастомізація, сертифікація)

10. Схожі рішення
    Картки інших мобільних рішень (швидка, штаб, броньовані тощо)

11. CTA-блок з формою
    «Отримати пропозицію за 1 день» — ім'я, телефон, тип задачі

12. Footer (як зараз)
```

Sticky справа на desktop у блоках 4–7: міні-картка «Замовити консультацію» з кнопкою і телефоном.

## 3. Що покажу візуально

Згенерую **3 рендерені напрями дизайну** (HTML-прототипи), щоб ви порівняли стилістику. У кожному напрямі буде показано:

- **Перший екран (Hero)** — повна композиція з фото авто, заголовком, УТП, CTA, бейджем партнера, стрічкою швидких фактів.
- **Блок «Призначення / сценарії»** — 3 картки використання з іконками.
- **Блок «Галерея з вкладками»** — згруповані фото з підписами замість однієї суцільної сітки.

Напрями відрізнятимуться характером (наприклад: суворо-індустріальний / чистий технологічний / преміум-корпоративний), але всі триматимуть бренд REFORM (темно-синій + жовтий акцент) і додадуть «еко»-ноту для цієї конкретної моделі.

Після того як виберете напрям — реалізую повну сторінку за цією структурою.

### Технічні деталі реалізації (для розробки)
- Окремий route `src/routes/mobile-solutions.eco-lab-nissan.tsx` (TanStack Start file-based routing) з власним `head()` (title, description, og:image = hero фото).
- Дизайн-токени (кольори REFORM, типографіка) — у `src/styles.css` як CSS-змінні в oklch, далі через семантичні Tailwind-класи.
- Галерея — компонент із Tabs (shadcn) + Dialog як lightbox.
- Форма CTA — серверна функція через `createServerFn` (надсилання заявки), валідація Zod.
- Структуровані дані JSON-LD `Product`/`Vehicle` для SEO.

**Наступний крок:** згенерувати 3 дизайн-напрями і показати вам на вибір.
